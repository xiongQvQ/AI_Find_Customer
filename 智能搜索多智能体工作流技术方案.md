# 智能搜索多智能体工作流技术方案

## 📋 项目概述

将现有的手动关键词搜索系统升级为智能化的多智能体工作流系统，实现用户自然语言需求到精准搜索结果的端到端自动化。

### 用户场景示例
- **输入**: "我想找卖数位板的公司，要求支持4K分辨率，价格1000-3000元，深圳地区"
- **输出**: 按匹配度排序的公司列表，每个结果包含AI评分和匹配原因

### 核心价值
- **用户体验**: 从手动填表到自然语言对话
- **搜索精度**: 实时AI评分，确保结果相关性
- **系统效率**: 多智能体并行协作，显著提升性能

## 🤖 多智能体角色设计

### 1. 🎯 OrchestratorAgent (总协调者)
**核心职责:**
- 工作流状态管理和Agent调度
- 错误处理和异常恢复
- 性能监控和资源分配

**技术实现:**
```python
class OrchestratorAgent:
    def __init__(self):
        self.workflow_state = WorkflowState()
        self.agent_registry = AgentRegistry()
        self.message_bus = MessageBus()
    
    async def process_user_request(self, user_input: str) -> SearchResult:
        # 1. 启动工作流
        workflow_id = self.start_workflow(user_input)
        
        # 2. 协调各Agent按序执行
        requirement = await self.coordinate_requirement_analysis(user_input)
        search_tasks = await self.coordinate_strategy_planning(requirement)
        raw_results = await self.coordinate_search_execution(search_tasks)
        scored_results = await self.coordinate_scoring(raw_results, requirement)
        final_results = await self.coordinate_result_optimization(scored_results)
        
        return final_results
```

### 3. 🔍 SearchStrategyAgent (搜索策略师)
**专业领域:** 搜索引擎优化、关键词生成、策略规划

**核心策略:**
1. **多维度关键词生成**: 产品词 + 规格词 + 地域词组合
2. **搜索策略多样化**: 精确搜索 + 语义搜索 + LinkedIn专业搜索
3. **动态策略调整**: 根据初步结果质量调整后续搜索

**实现逻辑:**
```python
class SearchStrategyAgent:
    def generate_search_tasks(self, requirement: RequirementSpec) -> List[SearchTask]:
        tasks = []
        
        # 策略1: 精确产品搜索
        exact_keywords = self.generate_exact_keywords(requirement)
        tasks.extend(self.create_general_search_tasks(exact_keywords))
        
        # 策略2: 语义扩展搜索
        semantic_keywords = self.generate_semantic_keywords(requirement)
        tasks.extend(self.create_semantic_search_tasks(semantic_keywords))
        
        # 策略3: LinkedIn专业搜索
        linkedin_queries = self.generate_linkedin_queries(requirement)
        tasks.extend(self.create_linkedin_search_tasks(linkedin_queries))
        
        # 策略4: 地域定向搜索
        if requirement.location:
            local_tasks = self.create_location_specific_tasks(requirement)
            tasks.extend(local_tasks)
        
        return self.optimize_task_priority(tasks)
```

### 4. ⚡ SearchExecutorAgent (搜索执行者)
**专业领域:** 高并发API调用、数据收集、实时处理

**核心特性:**
- 支持多种搜索源 (Serper API + LinkedIn + 自定义)
- 智能限流和错误恢复
- 实时数据清洗和去重

**并发执行架构:**
```python
class SearchExecutorAgent:
    def __init__(self):
        self.api_clients = {
            'serper': SerperClient(),
            'linkedin': LinkedInSearchClient()
        }
        self.rate_limiters = {api: RateLimiter() for api in self.api_clients}
    
    async def execute_search_tasks(self, tasks: List[SearchTask]) -> List[CompanyResult]:
        # 按API类型分组任务
        grouped_tasks = self.group_tasks_by_api(tasks)
        
        # 并发执行各组任务
        all_results = []
        for api_type, api_tasks in grouped_tasks.items():
            api_results = await self.execute_api_tasks(api_type, api_tasks)
            all_results.extend(api_results)
        
        # 实时去重和数据清洗
        unique_results = self.deduplicate_results(all_results)
        cleaned_results = self.clean_and_validate(unique_results)
        
        return cleaned_results
```

class ScoringAgent:
    async def score_companies(self, companies: List[CompanyResult], 
                            requirement: RequirementSpec) -> List[ScoredCompany]:
        
        # 并发评分所有公司
        scoring_tasks = [
            self.score_single_company(company, requirement) 
            for company in companies

async def parallel_search_and_score(self, tasks, requirement):

## 🛠️ 技术实现细节

### 消息总线设计
```python
class MessageBus:
    """Agent间通信的消息总线"""
    
    def __init__(self):
        self.subscribers = defaultdict(list)
        self.message_queue = asyncio.Queue()
    
    def subscribe(self, event_type: str, agent: BaseAgent):
        self.subscribers[event_type].append(agent)
    
    async def publish(self, event: AgentEvent):
        for agent in self.subscribers[event.type]:
            await agent.handle_event(event)
```

### Agent基类设计
```python
class BaseAgent(ABC):
    def __init__(self, agent_id: str):
        self.agent_id = agent_id
        self.message_bus = None
        self.state = AgentState.IDLE
    
    @abstractmethod
    async def process(self, input_data: Any) -> Any:
        pass
    
    async def handle_event(self, event: AgentEvent):
        # 处理其他Agent发送的事件
        pass
    
    async def emit_event(self, event_type: str, data: Any):
        event = AgentEvent(type=event_type, data=data, source=self.agent_id)
        await self.message_bus.publish(event)
```

class WorkflowState:
